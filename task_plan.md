# Task Plan: 协作白板功能增强与性能优化

## 目标
- 补齐仪表盘与白板核心闭环：轻量列表、缩略图、最近访问、收藏、共享入口、只读角色、版本历史。
- 修复当前“设置项存在但不生效”的问题：显示网格、吸附到网格。
- 压缩白板热路径的性能浪费：图表按需加载、减少重复序列化、减少重复光标上报、避免本地回写触发远端重放。

## 约束与假设
- 事实：当前仓库未包含 Supabase migration 目录，也未提供 `board_collaborators` / `board_snapshots` 等表定义。
- 假设：线上 Supabase 当前至少已有 `boards`、`profiles` 两张表，且现有分享链接机制可继续访问白板。
- 策略：新增共享角色与版本历史时，前端优先支持本地持久化；若可选 Supabase 表存在，则自动启用远端能力；若不存在，则优雅降级而不阻塞当前功能。
- 注意：仓库存在未跟踪文件 `CollabBoard_Defense.pptx`、`build-ppt.cjs`、`ppt-workspace/`，本任务不改动它们。

## 分阶段任务
1. 规划与数据层
   - 创建/更新计划文件
   - 明确本地元数据存储结构
   - 设计可选 Supabase 协作/快照表的兼容方案
2. 仪表盘增强
   - 列表查询只取摘要字段
   - 新增本地白板元数据 store（缩略图、最近访问、收藏、共享缓存、角色）
   - 仪表盘展示“我的白板 / 协作白板”
3. 白板增强
   - 实现网格与吸附
   - 分享弹窗支持角色选择
   - 只读模式与 UI 权限收敛
   - 版本历史/快照
4. 性能优化
   - 图表组件懒加载
   - 合并保存/同步链路
   - 避免自己发送的数据被重新应用
   - 去掉重复的光标同步
5. 验证
   - 补单测
   - 运行 lint / test / build

## 当前状态
- 已完成：全部 5 个阶段已落地。
- 已验证：`npm run lint`、`npm test -- --run`、`npm run build`、`npm run test:e2e` 全部通过。
- 剩余注意项：`vendor-charts` 与 `vendor-antd` 仍然偏大，但图表面板已经改成按需加载，首屏压力已明显收敛。

## 校验点
- 仪表盘列表不再依赖 `boards.data`
- 共享白板至少可通过本地缓存重新进入
- 只读链接无法在前端执行编辑操作
- 白板修改后不会重复触发本地-远端-本地回放
- `npm run lint`、`npm test -- --run`、`npm run build`、`npm run test:e2e` 通过

## 风险
- 如果线上未配置共享/快照相关表，跨设备共享角色与远端版本历史只能部分生效。
- 版本历史若完全走本地存储，需要控制容量上限，避免 localStorage 膨胀。
